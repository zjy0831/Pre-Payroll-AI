# A1-A5 功能分析

## 整体判断

A1 到 A5 并不是 5 个孤立功能，而是一条完整的薪资风险控制链路。

- `A1-A4` 聚焦于薪资计算前校验
- `A5` 聚焦于薪资计算后、审批前的异常拦截

这组能力的真正产品主线，不是“AI 自动算薪”，而是“AI 帮助识别异常，人工完成审核与决策”。这个定位更适合企业薪资场景，因为它更容易满足审计、合规、责任归属和客户可配置化要求。

## 分项分析

### A1 考勤与加班异常检测

A1 是这组功能里最典型、也最适合优先落地的基础能力。它主要解决薪资截止前的考勤核对问题，包括漏打卡、超时加班、加班未审批、打卡与请假重叠、法定假日或休息日打卡等场景。

它的优势很明确：

- 数据来源清晰，主要依赖考勤和请假模块
- 异常规则容易解释，HR 容易理解和接受
- 可以明显减少 payroll cutoff 前的人工排查时间
- 产出形式天然适合做异常清单和待办列表

主要风险在于：

- 各国家、各客户的加班规则差异较大，必须支持客户级阈值配置
- 如果底层考勤数据质量不高，用户会很快失去对结果的信任
- 产品文案上要区分“异常”“疑似违规”“需要复核”，不能混为一谈

结论：A1 很适合作为第一批上线功能，也是后续异常中心的入口能力。

### A2 请假余额与薪资影响检查

A2 直接针对“请假影响薪资”这个高频错误源。相比 A1，它更接近薪资结果本身，因为它关注的是请假状态、余额、兑现资格、按比例计算是否会直接影响当期工资。

它的价值体现在：

- 能把请假审批状态和薪资影响直接关联起来
- 能帮助 HR 在算薪前处理待审批请假，减少返工
- 能降低员工对扣薪、无薪假、假期兑现等问题的投诉和争议

主要风险在于：

- 对客户政策配置依赖较高，例如 encashment、carry-over、auto-deduct 等规则必须完整
- 文档中“异常请假模式”这部分更偏行为洞察，和“薪资影响检查”的主线不完全一致
- 如果范围不控制，A2 容易从 payroll checker 扩展成 leave analytics，导致功能边界变大

结论：A2 应该和 A1 一起看，二者天然适合整合成统一的 pre-payroll exception workflow。

### A3 临时支付与扣款校验

A3 处理的是手工输入的薪资项目，例如奖金、佣金、报销、临时扣款等。这类数据虽然不一定像考勤那样高频，但单笔金额往往更大，一旦出错，财务风险更高。

它的价值主要在于：

- 能识别重复录入、超政策上限、缺少审批、统计异常值等问题
- 对 variable pay 较多的客户非常有吸引力
- 适合和审批流、风险等级一起设计

限制也比较明显：

- 统计异常判断依赖 3 个月以上历史数据
- 对公司政策配置的完整性要求高
- 不同 ad-hoc item 的业务语义差异大，规则抽象会更复杂

结论：A3 是高价值的第二阶段能力，但不如 A1/A2 基础，也不如 A5 关键。

### A4 薪资波动预测

A4 已经从“校验错误”延伸到“管理洞察”。它在正式算薪前给出成本预估、部门差异、成本中心下钻和 AI 叙述总结，更适合管理层和财务视角。

它的亮点在于：

- 把产品从异常检测提升到成本分析层面
- 对预算敏感月份、奖金月、组织调整期更有价值
- 展示形式容易做成 dashboard，适合对外演示

但它也存在几个问题：

- 它强依赖 A1/A2 数据管道和成本中心配置
- 成本波动不一定等于风险，误报的业务解释成本较高
- 从优先级上看，它更偏战略分析，不是最先要做的保底能力

结论：A4 适合作为后续增强项，不应占用首批 MVP 的核心资源。

### A5 薪资计算后异常检查

A5 是整组里最关键的最终拦截点。它不是在算薪前做预警，而是在算薪完成后、审批和发放前进行最终风险扫描。这个位置非常重要，因为它直接决定能否挡住高风险错误进入真实付款。

它的核心优势包括：

- 位于最后一道审批门之前，风险控制价值最高
- 覆盖净薪差异、同岗异常、缺失扣款、异常补贴、税额变化、新增薪资项、扣款突变等高风险场景
- 人机协同逻辑清楚：AI 标记，人工接受/调查/覆盖，每一步都要留痕
- 文档里明确提到已有 `Payroll_Anomalies_V3` 原型，说明落地成熟度高于纯概念方案

它的挑战主要在于：

- 依赖薪资历史数据和前序数据管道
- 同时使用规则和统计方法，误报率控制会决定用户是否真正信任它
- 审批门禁逻辑必须设计得足够清晰，否则会影响处理效率

结论：A5 应该被视为和 A1/A2 同等级的重要能力，尤其适合作为对外展示的核心卖点。

## 优先级建议

如果按“业务价值 + 落地难度 + 依赖关系”综合排序，建议顺序如下：

1. A1 + A2
2. A5
3. A3
4. A4

这样排序的原因是：

- A1 和 A2 最贴近日常操作痛点，依赖更少，也最容易形成第一版异常处理闭环
- A5 的风控价值最高，但更依赖前置数据和历史数据沉淀
- A3 是高价值专项校验，适合在基础流程稳定后补强
- A4 更偏管理分析层，适合放在后续版本提升产品战略高度

## 产品方向建议

从产品包装角度，这组能力最合适的表述不是“Payroll AI 自动化”，而是：

`AI 驱动的薪资异常管理与审批控制`

这个方向更稳，原因有三点：

- 企业客户更容易接受“AI 辅助审核”而不是“AI 替代决策”
- 更容易满足合规、审计、责任留痕要求
- 更利于和现有 payroll 流程对接，不需要重构客户认知

## 最终结论

这份文档里，真正的核心价值功能是 A1、A2、A5。

- `A1` 负责从考勤与加班维度提前发现异常
- `A2` 负责从请假与扣薪关系维度提前发现异常
- `A5` 负责在最终审批前做最后一道风险拦截

A3 适合作为第二阶段的手工项校验增强，A4 适合作为更偏管理分析的战略增强项。

如果要形成清晰的对外产品故事，这 5 项最好的组合逻辑是：

`先在算薪前减少错误，再在审批前拦截残余异常，并保证全过程可审计。`
